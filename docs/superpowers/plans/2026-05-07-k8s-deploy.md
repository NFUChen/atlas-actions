# K8s Deploy Workflows — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create two reusable GitHub Actions workflows for deploying to Kubernetes — one for raw manifests (`kubectl apply`) and one for Helm charts (`helm upgrade --install`) — plus unified documentation.

**Architecture:** Two standalone `workflow_call` workflows, each with kubeconfig-based auth and optional WireGuard VPN. No shared composite actions or cross-workflow dependencies. Each workflow is fully self-contained.

**Tech Stack:** GitHub Actions YAML, kubectl, Helm

**Spec:** `docs/superpowers/specs/2026-05-07-k8s-deploy-design.md`

| `.github/workflows/k8s-deploy-helm.yml` | Create | Deploy Helm charts via `helm upgrade --install` |
| `docs/K8S.md` | Create | Usage documentation for both workflows |

- [ ] **Step 1: Create the workflow file**

Create `.github/workflows/k8s-deploy-manifest.yml` with this exact content:

```yaml
name: K8s Deploy (Manifest)

on:
  workflow_call:
    inputs:
      manifest-path:
        description: "Path to manifest file or directory (relative to repo root)"
        required: true
        type: string
      namespace:
        description: "Target Kubernetes namespace"
        required: false
        type: string
        default: "default"
      kubectl-version:
        description: "kubectl version to install (e.g. v1.31.0)"
        required: false
        type: string
        default: "latest"
    secrets:
      kubeconfig:
        description: "Full kubeconfig file content (base64 encoded)"
        required: true
      wg-config-file:
        description: "WireGuard configuration file content for VPN tunnel"
        required: false

jobs:
  deploy:
    name: Deploy Manifests
    runs-on: ubuntu-latest
    steps:
      - name: Checkout caller repo
        uses: actions/checkout@v4

      - name: Set up WireGuard connection
        if: ${{ env.WG_CONFIG_FILE != '' }}
        uses: niklaskeerl/easy-wireguard-action@v2
        env:
          WG_CONFIG_FILE: ${{ secrets.wg-config-file }}
        with:
          WG_CONFIG_FILE: ${{ secrets.wg-config-file }}

      - name: Set up kubectl
        uses: azure/setup-kubectl@v4
        with:
          version: ${{ inputs.kubectl-version }}

      - name: Write kubeconfig
        env:
          KUBECONFIG_B64: ${{ secrets.kubeconfig }}
        run: |
          mkdir -p ~/.kube
          echo "$KUBECONFIG_B64" | base64 -d > ~/.kube/config
          chmod 600 ~/.kube/config

      - name: Verify cluster connectivity
        run: |
          if ! kubectl cluster-info 2>&1; then
            echo "::error::Cannot connect to Kubernetes cluster. Check kubeconfig and network connectivity."
            exit 1
          fi
          echo "Cluster connectivity check passed."

      - name: Apply manifests
        id: apply
        env:
          MANIFEST_PATH: ${{ inputs.manifest-path }}
          NAMESPACE: ${{ inputs.namespace }}
        run: |
          OUTPUT=$(kubectl apply -f "$MANIFEST_PATH" -n "$NAMESPACE" 2>&1)
          echo "$OUTPUT"
          {
            echo "result<<EOF"
            echo "$OUTPUT"
            echo "EOF"
          } >> "$GITHUB_OUTPUT"

      - name: Write job summary
        env:
          MANIFEST_PATH: ${{ inputs.manifest-path }}
          NAMESPACE: ${{ inputs.namespace }}
          APPLY_OUTPUT: ${{ steps.apply.outputs.result }}
        run: |
          {
            echo "## K8s Deploy (Manifest)"
            echo ""
            echo "**Namespace**: \`${NAMESPACE}\`"
            echo "**Manifest path**: \`${MANIFEST_PATH}\`"
            echo ""
            echo "### Applied Resources"
            echo ""
            echo '```'
            echo "$APPLY_OUTPUT"
            echo '```'
          } >> "$GITHUB_STEP_SUMMARY"
```

### Task 2: Create the Helm deploy workflow

**Files:**
- Create: `.github/workflows/k8s-deploy-helm.yml`

- [ ] **Step 1: Create the workflow file**

Create `.github/workflows/k8s-deploy-helm.yml` with this exact content:

```yaml
name: K8s Deploy (Helm)

on:
  workflow_call:
    inputs:
      chart-path:
        description: "Path to Helm chart directory (relative to repo root)"
        required: true
        type: string
      release-name:
        description: "Helm release name"
        required: true
        type: string
      namespace:
        description: "Target Kubernetes namespace"
        required: false
        type: string
        default: "default"
      values-file:
        description: "Path to values file (relative to repo root)"
        required: false
        type: string
        default: ""
      set-values:
        description: "Newline-separated key=value pairs passed as --set flags"
        required: false
        type: string
        default: ""
      helm-version:
        description: "Helm version to install (e.g. v3.16.0)"
        required: false
        type: string
        default: "latest"
    secrets:
      kubeconfig:
        description: "Full kubeconfig file content (base64 encoded)"
        required: true
      wg-config-file:
        description: "WireGuard configuration file content for VPN tunnel"
        required: false

jobs:
  deploy:
    name: Deploy Helm Chart
    runs-on: ubuntu-latest
    steps:
      - name: Checkout caller repo
        uses: actions/checkout@v4

      - name: Set up WireGuard connection
        if: ${{ env.WG_CONFIG_FILE != '' }}
        uses: niklaskeerl/easy-wireguard-action@v2
        env:
          WG_CONFIG_FILE: ${{ secrets.wg-config-file }}
        with:
          WG_CONFIG_FILE: ${{ secrets.wg-config-file }}

      - name: Set up Helm
        uses: azure/setup-helm@v4
        with:
          version: ${{ inputs.helm-version }}

      - name: Write kubeconfig
        env:
          KUBECONFIG_B64: ${{ secrets.kubeconfig }}
        run: |
          mkdir -p ~/.kube
          echo "$KUBECONFIG_B64" | base64 -d > ~/.kube/config
          chmod 600 ~/.kube/config

      - name: Verify cluster connectivity
        run: |
          if ! kubectl cluster-info 2>&1; then
            echo "::error::Cannot connect to Kubernetes cluster. Check kubeconfig and network connectivity."
            exit 1
          fi
          echo "Cluster connectivity check passed."

      - name: Deploy with Helm
        id: helm
        env:
          CHART_PATH: ${{ inputs.chart-path }}
          RELEASE_NAME: ${{ inputs.release-name }}
          NAMESPACE: ${{ inputs.namespace }}
          VALUES_FILE: ${{ inputs.values-file }}
          SET_VALUES: ${{ inputs.set-values }}
        run: |
          HELM_ARGS=()
          if [ -n "$VALUES_FILE" ]; then
            HELM_ARGS+=(-f "$VALUES_FILE")
          fi
          if [ -n "$SET_VALUES" ]; then
            while IFS= read -r line; do
              line=$(echo "$line" | xargs)
              [ -z "$line" ] && continue
              HELM_ARGS+=(--set "$line")
            done <<< "$SET_VALUES"
          fi
          OUTPUT=$(helm upgrade --install "$RELEASE_NAME" "$CHART_PATH" \
            -n "$NAMESPACE" \
            --create-namespace \
            "${HELM_ARGS[@]}" 2>&1)
          echo "$OUTPUT"
          {
            echo "result<<EOF"
            echo "$OUTPUT"
            echo "EOF"
          } >> "$GITHUB_OUTPUT"

      - name: Write job summary
        env:
          CHART_PATH: ${{ inputs.chart-path }}
          RELEASE_NAME: ${{ inputs.release-name }}
          NAMESPACE: ${{ inputs.namespace }}
          VALUES_FILE: ${{ inputs.values-file }}
          HELM_OUTPUT: ${{ steps.helm.outputs.result }}
        run: |
          {
            echo "## K8s Deploy (Helm)"
            echo ""
            echo "**Release**: \`${RELEASE_NAME}\`"
            echo "**Namespace**: \`${NAMESPACE}\`"
            echo "**Chart path**: \`${CHART_PATH}\`"
            if [ -n "$VALUES_FILE" ]; then
              echo "**Values file**: \`${VALUES_FILE}\`"
            fi
            echo ""
            echo "### Helm Output"
            echo ""
            echo '```'
            echo "$HELM_OUTPUT"
            echo '```'
          } >> "$GITHUB_STEP_SUMMARY"
```

# K8s Actions

Reusable GitHub Actions workflows for deploying to Kubernetes clusters. One workflow per deployment method.

## Workflows

| Workflow | Method | Description |
|----------|--------|-------------|
| `k8s-deploy-manifest.yml` | kubectl apply | Deploy raw K8s manifest files or directories |
| `k8s-deploy-helm.yml` | helm upgrade --install | Deploy Helm charts with values and overrides |

## Prerequisites

- A Kubernetes cluster accessible from GitHub Actions runners (or via WireGuard VPN)
- A base64-encoded kubeconfig stored as a repo secret
- Manifest files or a Helm chart in your repo

### Encoding kubeconfig

```bash
cat ~/.kube/config | base64 | pbcopy  # macOS
cat ~/.kube/config | base64 -w 0      # Linux
```

Store the output as a repo secret named `KUBECONFIG`.

## Usage

### Manifest Deploy

```yaml
name: Deploy

on:
  push:
    branches: [main]

jobs:
  deploy:
    uses: NFUChen/cloud-actions/.github/workflows/k8s-deploy-manifest.yml@main
    with:
      manifest-path: "k8s/"
      namespace: production
    secrets:
      kubeconfig: ${{ secrets.KUBECONFIG }}
```

### Manifest Deploy with VPN

```yaml
jobs:
  deploy:
    uses: NFUChen/cloud-actions/.github/workflows/k8s-deploy-manifest.yml@main
    with:
      manifest-path: "k8s/deployment.yaml"
      namespace: staging
    secrets:
      kubeconfig: ${{ secrets.KUBECONFIG }}
      wg-config-file: ${{ secrets.WG_CONFIG_FILE }}
```

### Helm Deploy

```yaml
jobs:
  deploy:
    uses: NFUChen/cloud-actions/.github/workflows/k8s-deploy-helm.yml@main
    with:
      chart-path: "charts/myapp"
      release-name: myapp
      namespace: production
      values-file: "charts/myapp/values-prod.yaml"
      set-values: |
        image.tag=${{ github.sha }}
        replicas=3
    secrets:
      kubeconfig: ${{ secrets.KUBECONFIG }}
      wg-config-file: ${{ secrets.WG_CONFIG_FILE }}
```

### Helm Deploy — Minimal

```yaml
jobs:
  deploy:
    uses: NFUChen/cloud-actions/.github/workflows/k8s-deploy-helm.yml@main
    with:
      chart-path: "charts/myapp"
      release-name: myapp
    secrets:
      kubeconfig: ${{ secrets.KUBECONFIG }}
```

## Inputs

### Manifest Workflow

| Name | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| `manifest-path` | string | yes | — | Path to manifest file or directory |
| `namespace` | string | no | `default` | Target K8s namespace |
| `kubectl-version` | string | no | `latest` | kubectl version to install |

### Helm Workflow

| Name | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| `chart-path` | string | yes | — | Path to Helm chart directory |
| `release-name` | string | yes | — | Helm release name |
| `namespace` | string | no | `default` | Target K8s namespace |
| `values-file` | string | no | `""` | Path to values file |
| `set-values` | string | no | `""` | Newline-separated `key=value` pairs for `--set` |
| `helm-version` | string | no | `latest` | Helm version to install |

## Secrets

Both workflows use the same secrets:

| Name | Required | Description |
|------|----------|-------------|
| `kubeconfig` | yes | Full kubeconfig file content (base64 encoded) |
| `wg-config-file` | no | WireGuard config file content for VPN tunnel |

## How It Works

```
Push to main
  └─► k8s-deploy-manifest runs
        ├─ WireGuard VPN (if configured)
        ├─ kubectl cluster-info (connectivity check)
        ├─ kubectl apply -f <path> -n <namespace>
        └─ writes result to job summary

  └─► k8s-deploy-helm runs
        ├─ WireGuard VPN (if configured)
        ├─ kubectl cluster-info (connectivity check)
        ├─ helm upgrade --install <release> <chart> -n <namespace>
        └─ writes result to job summary
```

## Pinning Tool Versions

Both workflows accept a version input for reproducibility:

```yaml
with:
  kubectl-version: "v1.31.0"   # manifest workflow
  helm-version: "v3.16.0"      # helm workflow
```

Default is `latest` for both.

## Helm Values

### Via values file

```yaml
with:
  values-file: "charts/myapp/values-prod.yaml"
```

### Via --set overrides

One `key=value` pair per line:

```yaml
with:
  set-values: |
    image.tag=abc1234
    replicas=3
    ingress.enabled=true
```

### Both together

```yaml
with:
  values-file: "charts/myapp/values-prod.yaml"
  set-values: |
    image.tag=${{ github.sha }}
```

`--set` values override values file entries, matching standard Helm behavior.
